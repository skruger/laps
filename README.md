# Lan Address Publishing Service

Receive requests from computers to update their IPv6 address in DNS. This is a simple local
only service. Use it at your own risk and always issue credentials with minimal permissions.

## Configuration

This is currently designed for AWS route53 but could be adapted to other providers.

The configuration file is in YAML format and looks like this:

```yaml
listen_port: 8080

domain: "shaunkruger.com"
r53_zone_id: ""

aws_access_key_id: ""
aws_secret_key: ""

clients:
  - preshared_key: "abcd1234"
    hostname: "lapstest.home.shaunkruger.com"
```

Each client in the list needs a preshared_key and a hostname. The clients are checked in order
and once it finds a preshared_key and hostname combination that passes authentication it will
update route53 and return a response.

The configuration file is read from /etc/laps/config.yaml by default but you can specify
a different filename with the `LAPS_CONFIG` environment variable.

## Creating the request
To update a record the client needs to post this JSON to the server POST the following to `/update`:

```json
{
  "hostname": "lapstest.home.shaunkruger.com",
  "ipv6_addr": "2605:a601:806b:d721:beef:f00d:db26:ae1",
  "ipv4_addr": "192.168.1.100",
  "timestamp": 1776463209,
  "signature": "e014859b286a668d4d679c2eb2d7f56db379b53328195de8264c71687cf8f8e8"
}
```

The signature is generated by generating the sha256 sum of the string
`<hostname>|<ipv6_addr>|<ipv4_addr|<timestamp>|<preshared_key>`.
